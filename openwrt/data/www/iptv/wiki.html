工作原理

运营商 IPTV 多数使用专用 IPTV 口、DHCP/认证、IGMP 加组和 UDP/RTP 组播。普通播放器无法直接知道所有频道地址，所以本工具让旁路由监听 IPTV 盒子的请求，提取频道组播地址，再生成家庭内网播放器可用的 M3U 节目单。

核心链路是：光猫 IPTV 口 -> 旁路由 IPTV 网口 -> 二层桥/转发 -> USB 网口 -> IPTV 盒子。旁路由优先保证盒子正常通信，再旁路分析 DHCP、IGMP、HTTP/API、UDP/RTP 组播。

页面当前按日常流程组织：先看状态和访问地址，再管理节目表，再处理抓包和 VSP，最后是高级网络和机顶盒伪装。

设备连接方式

  • eth1：推荐接光猫 IPTV 口，作为 IPTV 数据入口。
  • eth2：推荐 USB 网口，接运营商 IPTV 盒子，用于抓包和盒子直通。
  • br-iptv：IPTV 专用桥，不承担家庭 LAN 网关。
  • /etc/iptv-capture-iface：记录抓包口，默认 eth2。

网络模式

“抓包配套：网络模式与网口”区域主要是给抓包流程使用。先确认 eth1 接光猫 IPTV 口、eth2 或 USB 网口接 IPTV 盒子，再切换模式。

直通/抓包模式

优先保证 IPTV 盒子像直连光猫一样工作。适合正常使用盒子、抓取开机配置、抓原始组播。此模式下通常关闭或弱化 udpxy。

udpxy 单播模式

把 UDP 组播转换成 HTTP 单播，适合电视 App、Emby、VLC 等播放器使用。切换模式可能短暂影响播放，所以页面会提示确认。

组播绕开代理

IPTV 组播流量不应经过 Nikki/OpenClash。页面顶部状态和“确认组播不走代理”用于检查 239.0.0.0/8 是否保持直连。

抓包与频道发现

抓开机包 / VSP 完整频道表

用于盒子从断电开机到进入首页/直播页期间的流量分析。推荐流程：关闭盒子 -> 点击“开始抓开机包” -> 给盒子上电并进入直播/频道列表 -> 点击“停止开机包” -> 点击“分析开机包”。

这个流程会尽量从明文 HTTP/API 中提取配置地址、频道接口、EPG、组播地址、Cookie、JSESSIONID 和 VSP 请求信息。如果运营商使用 HTTPS 或私有加密，可能只能看到域名和 IP，无法直接还原完整节目单。

切台手动抓频道

用于补缺失频道。点击“开始切台抓频道”后，在 IPTV 盒子上逐个切到缺失频道，每个频道停留 3-5 秒。右侧状态栏会实时显示是否运行、抓包口、已抓到的组播地址数量、最新地址、频道名候选和台标候选。

点击“停止并补入节目单”后，系统会分析本次出现过的组播地址：已存在的源不会重复添加，新源会补入基础节目单，并尝试用明文识别到的频道名/台标、本地台标库、在线台标源和在线 EPG 图标补全信息。

组播盲扫

  • “组播盲扫”：只在抓不到包时小范围使用，例如 239.254.10.1-255:5000。盲扫可能产生较多探测流量，不建议大范围长期运行。
  • 盲扫只能生成候选地址，不能保证频道名称准确。优先使用 VSP 或切台抓频道获取名称和地址。

VSP 获取与按规则合并

“VSP获取并单独保存”会利用开机包里识别到的 VSP 会话，先调用 QueryAllChannel 获取频道清单，再逐个调用 PlayChannel 换取组播播放地址。这个过程可能较慢，右侧状态栏会显示频道总数、当前进度、当前频道、成功/失败/跳过数量和最近地址。

VSP 获取不会直接修改正式节目单，而是单独保存候选文件：

  • /www/vsp.m3u：VSP 基础候选节目单。
  • /www/vsp.tsv：结构化候选清单，供后续按规则合并。

确认 VSP 候选数量和名称正常后，到“节目表在线编辑”区域点击“按规则合并”。当前合并顺序为：4K/超高清 -> CCTV-1 到 CCTV-N -> 热门省级卫视 -> 地方卫视 -> VSP 其他频道 -> 原节目单剩余频道 -> 国外频道。CCTV-4K 留在 4K/超高清分类里；CCTV 内部按频道号升序排列，同序号高清源优先作为线路 1；同名但地址不同才会保留为多线路。

节目表、台标和历史

面向播放器的正式节目单主要有两份：基础节目单 new.m3u 和带台标节目单 new-logo.m3u。访问地址区域默认只保留本地直播源、本地带台标和本地 EPG。

在线编辑器会显示当前正在编辑的文件名和节目数量。统计规则是读取文本里的 #EXTINF 行数，所以手动编辑时数量会即时变化。

每次保存会生成历史备份，保留数量可设置为 3-10 版，修改失败可以选择历史版本恢复。M3U 一个频道通常由 #EXTINF 行和下一行播放地址组成，调整顺序时两行要一起移动。

“可选删除失效源”会先检测基础节目单里的播放地址，再列出可逐条删除的失效源，不会直接批量误删。

“一键增加台标”会以 new.m3u 为基础，先复用旁路由本地台标库，再从在线台标源和在线 EPG 图标补齐，结果写入 new-logo.m3u，不改动基础节目单。“一键去除台标”会移除 tvg-logo 字段，结果写入 new.m3u。

本地 EPG、在线源和台标补全

“本地 EPG 管理”用于把多个在线 XMLTV/EPG 源汇总到旁路由本地，生成 /www/epg.xml.gz 和兼容地址 /www/t.xml.gz。播放器和 Emby 建议优先使用旁路由本地 EPG：http://路由器IP:8080/epg.xml.gz。

在线 EPG 源每行一个，支持 .xml 和 .xml.gz。系统会按当前 new.m3u 的频道名过滤，只保留节目单里实际用得到的频道和节目，避免把外部大 EPG 全量塞进路由器。

在线台标源每行一个模板，必须包含 {name}。系统会用频道原名和归一化后的频道名分别替换 {name} 尝试下载。默认源是 GitHub fanmingming 台标库。

频道别名区域保留兼容规则，例如 CCTV$1 => regex:/^CCTV[-\s]*(\d{1,2}(\s*P(LUS)?|[K\+])?)(?![\s-]*(美洲|欧洲)).*/i。由于 OpenWrt busybox 不完整支持 PCRE 负向判断，CCTV 系列实际由插件内置兼容规则处理，避免正则不兼容导致更新失败。

“一站式更新”会依次执行：生成本地 EPG、从 EPG 里提取 <icon src> 台标、从在线台标源补缺、为仍缺失台标的频道生成本地文字兜底台标，最后重建 new-logo.m3u。

定时任务默认启用，时间窗 00:00-23:59，间隔 6 小时。它只更新 EPG 和台标，不切换网口、不重启网络、不重新排序节目单。保存天数用于保留本地 EPG 历史备份，防止在线源异常时无法回滚。

可选外部工具箱

本插件已不依赖 php-epg。新机器只安装本插件即可完成 IPTV 网络修复、节目单、udpxy、台标缓存和本地 EPG 输出。

如果你已经有群晖 php-epg，可把它保留为可选素材仓库：将主机 IP 写入 /etc/soso-iptv-epg-host 后，“同步外部工具箱台标”会读取 /data/icon/，本地 EPG 生成器也会把该主机的 t.xml.gz 追加为一个可选上游源。

没有配置 /etc/soso-iptv-epg-host 时，插件不会主动访问外部工具箱，也不会把 NAS 在线状态当作本机 EPG 正常。

日志与跨客户端查看

管理页底部有两个独立日志区域，适合手机操作、电脑查看结果。日志保存在旁路由本地 /tmp/iptv/trash/，不同浏览器、不同设备打开页面时看到的是同一份服务端日志。

抓包日志

抓包日志用于查看开机包、VSP 获取、切台抓频道、抓包临时数据清理等流程。对应文件是 /tmp/iptv/trash/capture-ui-log.txt。如果你用手机点击“VSP获取并单独保存”，电脑端打开页面后点击“查看抓包日志”即可看到同一份进度和最终结果。

抓包日志区域还会叠加显示当前 VSP 进度、候选节目单条数、最近失败频道，以及切台抓包已经发现的组播地址数量。

系统日志

系统日志用于查看普通管理操作，例如自检、网口设置、USB 抓包口调试、模式切换、保存频道、节目表操作、账号与配置修改等。对应文件是 /tmp/iptv/trash/ui-log.txt。

下载与清空

两个日志区域都提供“查看、下载、清空”按钮。下载会把当前服务端日志保存为 .txt 文件；清空只清对应日志文件，不删除 pcap 抓包、节目单、VSP 候选文件、主题素材或历史版本。

页面配置、主题和清理

后台 YAML 配置

/etc/soso-iptv-gateway.yaml 用于自定义页面标题、作者、说明、Logo 和访问地址。普通字段写法：title: IPTV 家庭网关；链接字段写法：- title: 名称 | url: 地址 | note: 备注。

主题和皮肤

右上角“主题”可以配置主题色、背景色、背景图、Logo 和面板透明度。背景色会联动面板、按钮、边框和输入框整体色调；透明度范围为 15%-100%。上传的背景图和 Logo 会持久化到 /www/iptv/assets/。

账号密码

右上角“账号”用于修改管理页用户名和密码。安装包默认用户名 admin、密码 admin。修改时会校验旧账号和旧密码，即使浏览器保存的旧登录状态过期，也可以通过正确旧密码完成更新。

运行时数据与清理

运行时目录统一为 /tmp/iptv，可清理临时文件统一放在 /tmp/iptv/trash。清理按钮只删除抓包、分析、候选列表等临时数据，不删除正式节目单、VSP 持久文件、主题素材或编辑历史。

机顶盒伪装

“IPTV 接口 / 机顶盒伪装”属于高级设置，作用在旁路由的 IPTV 专用入口上，通常对应接光猫 IPTV 口的网口，不是家庭 LAN，也不是普通上网 WAN。

它用于运营商 IPTV 侧需要识别机顶盒身份的环境，可配置 MAC 克隆、DHCP Hostname、Option 60、Option 61 或 PPPoE 账号密码。普通抓包、VSP 获取、切台补频道不需要频繁改这里。

改错可能导致 IPTV 侧拿不到地址或盒子无法播放。修改前建议先确认当前网口名称和接线。

常见问题与修复

  • USB 口 DOWN：检查 USB 网卡、网线、盒子电源和盒子网口。
  • 抓不到 IGMP：确认盒子正在播放直播频道，不是停在首页或点播页。
  • 组播走代理：检查 Nikki/OpenClash 路由规则，239.0.0.0/8 应绕开代理；也可直接点「网络与接口 → 修复被透明代理」一键强制直连。
  • Emby 不出图标：本插件已把台标统一指向旁路由本地 /www/iptv/logos/（带 ?v= 版本号），插件侧 URL 正常即为真 PNG。Emby 显示坏图基本是缓存了改之前的旧链接——解决：在 Emby 里对该 M3U 调谐器执行「刷新指南数据」，或移除后重新添加 M3U 源；必要时清理 Emby 图片缓存。换源后台标 URL 版本号已变，会强制重新抓取。少数本地台标库没有的频道会无图（非坏图）。
  • VSP 很慢：这是逐频道调用 PlayChannel 的正常现象，看右侧进度即可。如果长时间无进度，通常是 JSESSIONID 过期或开机包缺少 VSP 会话。
  • 只抓到几个频道：通常只是切台时抓到了当前播放/切换过的频道。要完整频道表优先用 VSP；补缺失频道用切台手动抓。
  • 节目表数量异常：在节目表在线编辑器里读取对应文件，查看“当前节目数量”，必要时从历史版本恢复。
  • 公网不可用：本工具默认仅内网使用，外部访问入口已移除。
