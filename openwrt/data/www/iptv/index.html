这套 IPTV 网关解决什么问题

运营商 IPTV 通常不是普通公网视频流，而是通过光猫 IPTV 口下发 DHCP、IGMP 和 UDP/RTP 组播。普通播放器无法直接跨内网使用这些组播地址，所以旁路由负责把运营商盒子侧看到的原始组播信息抓出来，整理成家庭播放器能使用的节目单。

当前设计优先保证运营商 IPTV 盒子能正常工作，再顺手抓取频道地址并生成节目单。也就是说，这套方案不是替代运营商盒子认证，而是让旁路由“旁听”盒子的频道请求。

设备连接方式

光猫 IPTV 口 接旁路由的 IPTV 入口网口，当前按 eth1 使用。

运营商 IPTV 盒子 接旁路由的 USB 网口，当前按 eth2 使用。页面状态里的“盒子侧 USB 抓包口”为 UP，才表示盒子侧链路已经接通。

家庭内网设备 继续通过原来的 LAN/Wi-Fi 访问节目单，例如播放器、电视、手机、电脑访问 http://路由器IP:8080/new-logo.m3u。

旁路由上的实际配置

br-iptv 是 IPTV 专用桥设备，包含 eth1 和 eth2。它不承担家庭 LAN 网关职责，只处理 IPTV 侧链路。

/etc/iptv-capture-iface 保存抓包口，默认是 eth2，也就是 USB 网口/盒子侧。

/etc/init.d/iptv-tc-bridge 使用 Linux tc mirred redirect 做 eth1 与 eth2 的二层互转，让盒子尽量像直连光猫 IPTV 口一样拿地址、发 IGMP、收组播。

udpxy 当前默认不启用，因为你的现阶段目标是保证盒子直通和节目单生成，不让旁路由主动抢 IPTV DHCP。后续如果确认运营商允许多终端或需要旁路由直接转单播，再启用更合适。

工作原理

盒子播放频道时，会先向 IPTV 网络发送 IGMP 加组请求，然后光猫 IPTV 口侧会出现对应的 UDP/RTP 组播流，例如 239.x.x.x:端口。

点击“开始抓原始组播”后，旁路由在 USB 盒子侧网口抓包，只记录抓包期间真正出现过的组播地址。同时会轻量监听盒子侧明文 HTTP/API 流量，尝试识别频道名称和 png/jpg/webp 台标资源。

点击“停止分析并自动补频道”后，系统会把本次抓到的地址和现有基础节目单 new.m3u 做差异比对。已经存在的频道不会重复写入；如果同时识别到频道名或台标，会直接更新现有 M3U 条目的 tvg-name、tvg-logo 和显示名称。

如果抓包没有直接拿到台标，但能识别到频道名，系统会继续从旁路由本地台标库、在线 EPG 图标和可选图片模板里尝试补全。外部工具箱只在手动配置主机后作为可选素材源。

新发现的地址会追加到 new.m3u 和 new.txt。如果识别到了频道名称和台标，新条目会按原有 #EXTINF 格式写入这些标注，并自动刷新 new-logo.m3u。

第一次设置流程

1. 先确认光猫 IPTV 口、旁路由 IPTV 入口、USB 网口、运营商盒子的线都接好。

2. 打开本页面，先看顶部状态卡：eth1、br-iptv 应该 UP；盒子开机并插好时 eth2 应该 UP。

3. 点“一键自检”，确认 tc 二层透传规则存在，EPG 能访问，节目单数量正常。

4. 在运营商盒子上播放直播频道，再点“播放异常分析”，应该能看到 DHCP、ARP 或 IGMP 活动。

5. 点“开始抓原始组播”，在盒子上切换缺失频道，每个频道停留 3-5 秒，再点“停止分析并自动补频道”。

6. 播放器使用 new-logo.m3u，如果频道顺序不满意，在“节目表在线编辑”里调整保存。

访问地址怎么用

本地直播源 给家里局域网播放器使用，优先选 new-logo.m3u，因为它包含台标信息。

本地 EPG 由旁路由汇总在线 EPG 源生成，播放器优先访问旁路由本地地址；外部 php-epg/NAS 只是可选追加源。

外部访问已移除：本工具只保留内网入口。公网反代不再作为推荐配置，减少误暴露管理页和节目单的风险。

每个地址右侧的“复制”按钮可以直接复制 URL，适合粘贴到播放器、电视 App 或远程设备里。

节目表在线编辑提示

M3U 一个频道通常由两行组成：第一行 #EXTINF 是频道名称、台标等信息，第二行是播放地址。调整顺序时这两行要一起移动。

保存 new.m3u 或 new-logo.m3u 时，系统会先备份旧文件。误改后可以在节目表编辑器的历史版本里一键恢复。

后台 YAML 配置

页面标题、作者、说明、Logo、访问地址列表由 /etc/soso-iptv-gateway.yaml 控制。适配不同设备时，优先改这个文件，不需要改 HTML。

普通字段写法：title: IPTV 家庭网关。链接字段写法：- title: 名称 | url: 地址 | note: 备注。以 # 开头的是备注。

主题和皮肤

主题颜色、背景颜色、背景图、Logo 和面板透明度会保存到路由器本地配置。上传的背景图和 Logo 会放在 /www/iptv/assets/，刷新页面后仍会生效。

如果背景图较花，可以把面板透明度调高；如果想让背景更明显，可以把透明度调低。

一键自检怎么看

网口状态：eth1 是光猫 IPTV 侧，eth2 是 USB 盒子侧，br-iptv 是桥。eth2 DOWN 通常表示盒子没开机、线没插好、USB 网卡未识别或盒子网口没有链路。

tc 二层透传规则：应看到 eth2 -> eth1 和 eth1 -> eth2 两个 redirect。缺一边时，盒子可能拿不到 IPTV 地址或无法收到回包。

节目单数量：用于快速判断节目单有没有被清空或生成异常。

EPG：检测旁路由本机生成的 /www/epg.xml.gz 能不能访问。

播放异常分析怎么看

它会临时抓 8 秒 DHCP、IGMP 和 ARP。看到 DHCP 说明盒子在请求或续租 IPTV 地址；看到 IGMP 说明盒子正在请求频道组播；只看到 ARP 或完全没有包时，通常要检查盒子是否真的在播放直播频道。

如果盒子本身都不能正常播放，先修复盒子直通链路；如果盒子能播但播放器不能播，再看节目单地址、Lucky 反代、播放器是否支持当前 URL 格式。

重要提醒

被动抓包不能凭空扫描全部频道，只能发现抓包期间盒子实际播放/切换过并产生流量的频道。缺几十个频道时，通常需要在盒子上逐个切到那些频道，或者从运营商/旧设备导出的节目单里合并。

组播数据不应该走 Nikki/OpenClash 代理。页面里的“确认组播不走代理”和“一键自检”就是为了确认 IPTV 路由仍在专用接口上。

台标库管理（本地）

台标已下载缓存到旁路由本地 /www/iptv/logos/，Emby 等本地软件从路由器本机读取，不依赖 NAS 在线，重启不丢。可逐个查看、上传替换或删除；替换后的台标会被「一键增加台标」保留。
上传建议：优先 PNG，透明背景最好；CCTV 这类横版台标可用 600×270，通用方形台标可用 512×512 或 256×256。jpg / webp / gif 会在后端尽量自动转换为 PNG，无法转换时会提示失败，避免 Emby 出现破图。

组播盲扫

盲扫只建议在抓不到包、找不到 EPG 文件时小范围使用。它会按固定段生成候选组播地址，可能产生较多探测流量，过大范围可能被运营商侧限制。

抓包补频道流程

1. 接线确认：光猫 IPTV 口接旁路由 IPTV 入口，运营商 IPTV 盒子接旁路由 USB 网口。当前盒子侧 USB 抓包口显示为 eth2 且状态为 UP 时再开始。

2. 开始抓包：点击“开始抓原始组播”。此时旁路由会从盒子侧网口抓取 IPTV 盒子的 IGMP 请求和 UDP 组播流量，并尝试从明文 HTTP/API 流量里识别频道名和台标 URL。

3. 盒子切台：在运营商 IPTV 盒子上连续切换你缺失的频道，每个频道停留 3-5 秒，让组播流真正出现。

4. 停止分析：点击“停止分析并自动补频道”。系统会把本次抓到的 239.x.x.x:端口 和现有节目单比对。

5. 自动写入：新地址会写入 new.m3u 和 new.txt，并自动重新生成 new-logo.m3u。已存在的地址不会重复添加；如果识别到台标或频道名，会更新现有条目的标注；没有抓到台标时会尝试从旁路由本地库、在线 EPG 图标和可选图片模板匹配。

6. 使用节目单：播放器优先使用 http://路由器IP:8080/new-logo.m3u。外部访问入口已移除，请仅在内网使用。

注意：这个功能不是全网扫描。只有盒子实际播放/切到过、并在抓包期间产生流量的频道，才能被发现并补进节目单。台标和频道名只能从明文资源中识别，HTTPS 加密或私有二进制协议无法直接解析。

IPTV 接口 / 机顶盒伪装

位置：这是高级设置，作用在旁路由的 IPTV 专用入口上，通常对应接光猫 IPTV 口的网口，不是家庭 LAN，也不是普通上网 WAN。

功能：当运营商 IPTV 网络需要识别机顶盒身份时，用这里配置 MAC 克隆、DHCP Hostname、Option 60、Option 61 或 PPPoE 账号密码，让旁路由在 IPTV 侧表现得更像原机顶盒。

使用场景：只有在盒子直通能播放、但旁路由自己拿不到 IPTV 地址，或运营商限制必须机顶盒认证时才需要改。普通抓包、VSP 获取、切台补频道不需要频繁动这里。

风险：改错可能导致 IPTV 侧拿不到地址或盒子无法播放。修改前建议先确认当前接线和网口名称，避免把家庭 LAN/WAN 配错。

保存后只修改 IPTV 专用接口配置，不主动修改家庭 LAN。PPPoE 仅用于运营商 IPTV 需要单独拨号登录的环境。
